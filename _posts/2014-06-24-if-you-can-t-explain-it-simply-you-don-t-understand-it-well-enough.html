Let's start with one of the easiest sorting algorithms: Bubble Sort.

Bubble Sort

Bubble sort works by repeatedly stepping through the list to be sorted, comparing each adjacent element and swapping them if they're in the wrong order.

Example

Input: 5, 3, 2, 8, 4, 1:

Iteration 1

5, 3, 2, 8, 4, 1
3, 5, 2, 8, 4, 1 - swapped
3, 2, 5, 8, 4, 1
3, 2, 5, 8, 4, 1 - swapped
3, 2, 5, 4, 8, 1 - swapped
3, 2, 5, 4, 1, 8

So now we have passed through the whole list once. Two things to note:

  • The largest number 8 is the largest number in the list and we can see it has 'bubbled' up to the end. This is how Bubble Sort gets its same
  • More importantly, remember we said 'remember that a swap occurred' above? Well, the algorithm carries on until we don't record any swapping i.e. until the list is sorted.

Let's do the same as before on our partially sorted list.

Iteration 2

3, 2, 5, 4, 1, 8 - swapped
2, 3, 5, 4, 1, 8
2, 3, 5, 4, 1, 8 - swapped
2, 3, 4, 5, 1, 8 - swapped
2, 3, 4, 1, 5, 8

Again, notice that the second largest element of 5 has bubbled up through the array. We recorded that a swap occurred so we continue applying the algorithm to the new array:

Iteration 3

2, 3, 4, 1, 5, 8
2, 3, 4, 1, 5, 8
2, 3, 1, 4, 5, 8 - swapped
2, 3, 1, 4, 5, 8
2, 3, 1, 4, 5, 8

We recorded that a swap occurred so we continue:

Iteration 4

2, 3, 1, 4, 5, 8
2, 1, 3, 4, 5, 8 - swapped
2, 1, 3, 4, 5, 8
2, 1, 3, 4, 5, 8
2, 1, 3, 4, 5, 8

We're nearly there, only the 2 and 1 are the wrong way now so we're one iteration away from having a sorted list:

Iteration 5

2, 1, 3, 4, 5, 8
1, 2, 3, 4, 5, 8 - swapped
1, 2, 3, 4, 5, 8
1, 2, 3, 4, 5, 8
1, 2, 3, 4, 5, 8

It looks sorted but a swap was recorded so the algorithm passes through the data once more

Iteration 6

1, 2, 3, 4, 5, 8
1, 2, 3, 4, 5, 8
1, 2, 3, 4, 5, 8
1, 2, 3, 4, 5, 8
1, 2, 3, 4, 5, 8

No swaps occurred so our list must be sorted.

Computational Complexity

Worst performance: O(n^2)
Best performance: O(n)
Average performance: O(n^2)
Space (worst): O(1) auxiliary

So the complexity isn't great. There are sorting algorithms with better worse and average complexities. Bubble sort is not practical for large data sets. One advantage that Bubble Sort has over Quicksort (not insertion sort) is that the algorithm has the ability to detect that the list is sorted built into the algorithm.
